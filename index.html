supported today

Three formats, done properly.

.pdf

Extracts real text from the page — layout, headings, and line breaks preserved as closely as a PDF allows, outputting clean .md.

.docx

Reads Word's actual structure — headings, bold, italics, links, and lists map straight to Markdown syntax as a .md file.

.txt

Cleans up spacing and line breaks so plain notes read like proper Markdown (.md), not a wall of text.

no server, no upload

Your file never leaves your machine.

Flashdown runs the entire conversion in your browser tab using JavaScript. There's no backend receiving your document — which also means it works offline once the page has loaded, and it's free to run at any scale.

questions

Good to know.

Is there a file size limit?

Only what your browser's memory allows — typically fine well past 50MB. Very large scanned PDFs may take a few seconds longer.

Will scanned (image-only) PDFs work?

Not yet. Flashdown extracts existing text layers — it doesn't do OCR on scanned images. That's on the roadmap.

Does it keep tables and images?

DOCX tables convert to Markdown tables. Images are referenced but not embedded, since Markdown can't carry binary data inline.

Is my document stored anywhere?

No. There's no server in the loop, so there's nothing to store. Close the tab and it's gone.

Does this work for PDF to MD, DOCX to MD, and TXT to MD?

Yes — Flashdown converts PDF, DOCX, and TXT files to clean .md (Markdown) format, entirely in your browser.
